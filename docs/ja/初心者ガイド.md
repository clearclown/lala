## Lalaとは何か？

Lalaは**テキストエディタ**です。テキストエディタとは、プログラムのコードや文章、設定ファイルなどのプレーンテキストを編集するためのソフトウェアです。

### Lalaの特徴

1. **GUIとCLIの両方に対応**
   - **GUI (Graphical User Interface)**: マウスで操作できるウィンドウ付きのエディタ
   - **CLI (Command Line Interface)**: ターミナルから使用できるコマンド

2. **軽量で高速**
   - Rustという高速なプログラミング言語で作られています
   - 大きなファイルもスムーズに扱えます

3. **マルチフォーマット対応**
   - Markdown、HTML、Mermaid、LaTeXなど様々な形式をプレビュー可能

## なぜLalaを使うのか？

### メリット

✅ **初心者に優しい**
- シンプルなインターフェース
- 直感的な操作
- 豊富なドキュメント

✅ **プログラマーに最適**
- シンタックスハイライト（コードに色が付く）
- 複数ファイルを同時に編集
- 高速な検索・置換機能

✅ **ドキュメント作成に便利**
- Markdownのリアルタイムプレビュー
- LaTeXの数式表示
- Mermaidで図表作成

✅ **無料でオープンソース**
- 誰でも無料で使える
- ソースコードが公開されている

## 基本的な概念

### テキストエディタとワープロの違い

| テキストエディタ | ワープロ（Word など） |
|-----------------|---------------------|
| プレーンテキスト | リッチテキスト |
| コード編集に適している | 文書作成に適している |
| ファイルサイズが小さい | ファイルサイズが大きい |
| プログラミング向け | ビジネス文書向け |

### ターミナルとは？

ターミナル（またはコマンドライン）は、文字だけでコンピュータを操作するためのツールです。

**Windows**: コマンドプロンプト、PowerShell、Windows Terminal
**Mac**: ターミナル.app
**Linux**: GNOME Terminal、Konsole など

### コマンドとは？

コマンドは、ターミナルに入力する命令です。

例：
```bash
lala --help      # Lalaのヘルプを表示
lala file.txt    # file.txtを開く
```

## 最初の一歩

### 1. インストール確認

ターミナルを開いて、以下のコマンドを入力：

```bash
lala --version
```

バージョン番号が表示されれば、インストール成功です！

**表示されない場合**:
- [インストールガイド](インストール.md)を参照
- PATHが正しく設定されているか確認

### 2. ヘルプを見る

```bash
lala --help
```

使用可能なコマンドの一覧が表示されます。

### 3. 最初のファイルを開く

#### GUIモードで開く

```bash
# 空のエディタを起動
lala

# 既存のファイルを開く
lala test.txt
```

ウィンドウが開いて、エディタが表示されます！

#### CLIモードでプレビュー

```bash
# Markdownファイルを作成
echo "# こんにちは、Lala！" > hello.md

# プレビュー
lala markdown hello.md
```

ターミナルに美しく整形されたMarkdownが表示されます。

## GUIモードの基本

### ウィンドウの構成

```
┌─────────────────────────────────────────┐
│  Lala - Modern Text Editor              │  ← タイトルバー
├──────────┬──────────────────────────────┤
│          │  タブ1 | タブ2 | タブ3       │  ← タブバー
│          ├──────────────────────────────┤
│  ファイル  │                              │
│  ツリー   │                              │
│          │      エディタエリア            │  ← テキスト編集エリア
│  📁 src  │                              │
│    main.rs                               │
│    lib.rs│                              │
│  📁 docs │                              │
│          │                              │
└──────────┴──────────────────────────────┘
```

### 基本操作

#### ファイルを開く

1. **ファイルツリーから**: 左側のファイルリストをクリック
2. **ドラッグ&ドロップ**: ファイルをウィンドウにドロップ（今後対応予定）

#### テキストを編集する

1. エディタエリアをクリック
2. 普通にキーボードで入力
3. **Ctrl+S**で保存（Mac: **Cmd+S**）

#### タブを管理する

- **新しいタブ**: ファイルツリーから別のファイルをクリック
- **タブを閉じる**: タブの×ボタンをクリック
- **タブを切り替え**: タブをクリック

### キーボードショートカット

| 操作 | Windows/Linux | Mac |
|------|--------------|-----|
| 保存 | Ctrl+S | Cmd+S |
| 元に戻す | Ctrl+Z | Cmd+Z |
| やり直し | Ctrl+Y | Cmd+Shift+Z |
| カーソル移動 | 矢印キー | 矢印キー |
| 行の先頭 | Home | Cmd+← |
| 行の末尾 | End | Cmd+→ |

## CLIモードの基本

### コマンド一覧

```bash
lala [オプション] [ファイル] [サブコマンド]
```

### 主要なサブコマンド

#### 1. markdown - Markdownプレビュー

```bash
lala markdown README.md
```

**使用例**:
- ブログ記事のプレビュー
- ドキュメントの確認
- READMEの整形確認

#### 2. html - HTMLプレビュー

```bash
lala html index.html
```

**使用例**:
- Webページのテキスト内容確認
- HTMLメールの内容確認

#### 3. mermaid - ダイアグラムプレビュー

```bash
lala mermaid diagram.mmd
```

**使用例**:
- フローチャートの表示
- シーケンス図の確認

#### 4. latex - LaTeXプレビュー

```bash
lala latex document.tex
```

**使用例**:
- 数式の確認
- 学術論文のプレビュー

#### 5. view - ファイル表示

```bash
# 行番号付きで表示
lala view -n src/main.rs

# 普通に表示
lala view README.md
```

**使用例**:
- ソースコードの確認
- ログファイルの閲覧

### オプション

```bash
# カラー出力を無効化
lala markdown README.md --no-color

# ヘルプを表示
lala markdown --help

# バージョンを表示
lala --version
```

## 実践例

### 例1: Markdownでドキュメントを書く

```bash
# 1. ファイルを作成
echo "# プロジェクト概要" > docs.md

# 2. GUIで編集
lala docs.md

# 3. プレビュー（別のターミナルで）
lala markdown docs.md

# 4. 保存してプレビュー更新
# GUIで Ctrl+S を押す
```

### 例2: コードを編集する

```bash
# 1. プロジェクトディレクトリを開く
lala /path/to/project

# 2. ファイルツリーからファイルを選択
# 3. コードを編集
# 4. Ctrl+S で保存
```

### 例3: Mermaidで図を作る

```bash
# 1. 図のファイルを作成
cat > flowchart.mmd << 'EOF'
graph TD
    A[開始] --> B{条件}
    B -->|Yes| C[処理1]
    B -->|No| D[処理2]
    C --> E[終了]
    D --> E
EOF

# 2. プレビュー
lala mermaid flowchart.mmd
```

## よくある質問

### Q1: ファイルが保存されているか確認したい

**A**: タブの名前の横に●が表示されていれば未保存です。**Ctrl+S**で保存してください。

### Q2: ターミナルでカラーが表示されない

**A**: ターミナルがカラーに対応していない可能性があります。

**解決方法**:
```bash
# カラーなしで表示
lala markdown README.md --no-color

# または、より良いターミナルを使う
# Windows: Windows Terminal
# Mac: iTerm2
# Linux: GNOME Terminal, Konsole
```

### Q3: GUIが起動しない

**A**: いくつかの原因が考えられます：

1. **Linuxの場合**: X11またはWaylandが必要
   ```bash
   echo $DISPLAY  # 何か表示されればOK
   ```

2. **依存ライブラリが不足**:
   ```bash
   # Ubuntu/Debian
   sudo apt install libxcb1 libxcb-render0

   # Arch Linux
   sudo pacman -S libxcb
   ```

### Q4: コマンドが見つからない

**A**: PATHが設定されていません。

```bash
# ファイルの場所を確認
which lala

# 何も表示されなければ、インストールを確認
cargo install lala

# またはPATHを追加（~/.bashrcまたは~/.zshrc）
export PATH="$HOME/.local/bin:$PATH"
```

### Q5: 文字化けする

**A**: ファイルのエンコーディングが異なる可能性があります。

**確認方法**:
```bash
file yourfile.txt
```

**対処方法**:
```bash
# UTF-8に変換
iconv -f SHIFT_JIS -t UTF-8 input.txt > output.txt
```
